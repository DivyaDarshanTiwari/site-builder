# 2.8 Assumptions and Dependencies

This section outlines the critical assumptions made during the design and development of the Site Builder platform, as well as the external dependencies required for the system to function correctly. If any assumptions prove false or dependencies fail, it will directly impact the project requirements and operational capabilities.

### Assumptions
1. **User Input Quality:** It is assumed that users can provide at least a basic, coherent text description (prompt) of the website they wish to build. While the internal AI prompt enhancement pipeline improves brief prompts, utterly nonsensical inputs will yield poor results.
2. **Modern Browser Usage:** It is assumed that end-users and the visitors to their generated sites will use modern web browsers (Chrome, Safari, Edge, Firefox) capable of rendering modern HTML5, executing ES6+ JavaScript, and parsing modern CSS (Tailwind styles).
3. **API Cost Projections:** It is assumed that the cost per token of the Google Gemini API will remain within projected margins, allowing the platform to maintain its current pricing models and credit system without operating at a loss.
4. **Acceptable Latency:** It is assumed that users are willing to tolerate a brief waiting period (typically 5 to 15 seconds) while the AI generates their website or processes revisions.
5. **Continuous Internet Connectivity:** The platform assumes a stable, high-speed internet connection from the user, as the application is entirely cloud-based and heavily reliant on continuous API communications.

### Dependencies
1. **Google Gemini API (Core Dependency):** The entire value proposition of the platform is strictly dependent on the availability, accuracy, and performance of the Google Gemini API (specifically the `gemini-2.5-flash` model). Any API outages, deprecations, or severe rate limitations will result in a total halt of core generation features.
2. **Tailwind CSS Infrastructure:** The websites generated by the AI rely exclusively on Tailwind CSS utility classes. Therefore, the rendered sites depend on the availability and rapid delivery of the Tailwind CSS script via CDN (`https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4`) injected into the generated HTML.
3. **Better-Auth Integration:** The platform's security and session management depend on the `better-auth` library and its active maintenance. Any undiscovered vulnerabilities within this library would compromise the application's user data and session integrity.
4. **Prisma ORM & PostgreSQL:** The backend infrastructure depends on the stability of the PostgreSQL database hosting provider and the reliable translation of queries through the Prisma ORM.
5. **OpenAI SDK / API Client:** The backend Node.js server depends on the `openai` npm package to interface with the AI models. (Note: Although the models are Gemini, the codebase utilizes the OpenAI SDK compatibility layer to interact with them).
